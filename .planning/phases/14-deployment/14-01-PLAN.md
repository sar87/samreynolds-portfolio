  <files>src/styles/variables.css, src/styles/global.css</files>
  <action>
**variables.css:** Replace fixed font-size tokens with fluid CSS clamp() values that scale from 320px to 1200px viewport. Keep the same min/max values but add viewport-relative scaling between them.

Replace these token values:
- `--font-size-xs`: `clamp(0.8125rem, 0.77rem + 0.22vw, 0.875rem)` (13px -> 14px)
- `--font-size-sm`: `clamp(0.875rem, 0.80rem + 0.34vw, 1rem)` (14px -> 16px)
- `--font-size-base`: `clamp(1rem, 0.91rem + 0.43vw, 1.125rem)` (16px -> 18px)
- `--font-size-lg`: `clamp(1.125rem, 1.01rem + 0.57vw, 1.25rem)` (18px -> 20px)
- `--font-size-xl`: `clamp(1.25rem, 1.07rem + 0.91vw, 1.5rem)` (20px -> 24px)
- `--font-size-2xl`: `clamp(1.5rem, 1.27rem + 1.14vw, 1.75rem)` (24px -> 28px)
- `--font-size-3xl`: `clamp(1.75rem, 1.30rem + 2.27vw, 2.25rem)` (28px -> 36px)
- `--font-size-4xl`: `clamp(2.25rem, 1.57rem + 3.41vw, 3rem)` (36px -> 48px)
- `--font-size-5xl`: `clamp(2.5rem, 1.59rem + 4.55vw, 3.5rem)` (40px -> 56px)
- `--font-size-6xl`: `clamp(3rem, 1.64rem + 6.82vw, 4.5rem)` (48px -> 72px)

Also add a mobile-specific content-padding variable to the existing variables, keeping desktop as default:
- Keep `--content-padding: var(--space-6)` as default (24px for desktop)
- Keep `--section-gap: var(--space-16)` as default

**global.css:** Add the following mobile foundations:

1. Change `body { min-height: 100vh }` to include dvh fallback:
```css
body {
  min-height: 100vh;
  min-height: 100dvh;
}
```

2. Add base touch target sizing for interactive elements (after the button reset section):
```css
/* Touch target sizing - WCAG 2.5.8 / Apple HIG 44px minimum */
@media (pointer: coarse) {
  button,
  a:not(.sr-only) {
    min-height: 44px;
    min-width: 44px;
  }

  /* Inline text links in prose content need padding, not min-width */
  .bio a,
  p a,
  li a {
    min-width: auto;
    padding-top: 8px;
    padding-bottom: 8px;
  }
}
```

3. Update the mobile `@media (max-width: 768px)` block to also reduce section-gap:
```css
@media (max-width: 768px) {
  :root {
    --content-padding: var(--space-4);
    --section-gap: var(--space-12);
  }
}

@media (max-width: 480px) {
  :root {
    --content-padding: var(--space-3);
    --section-gap: var(--space-10);
  }
}
```

Run `npm run build` to confirm no CSS syntax errors. Visually inspect at 320px in dev tools that:
1. Font sizes are smaller than desktop but still readable
2. Touch targets on buttons/links are at least 44px on touch devices

Typography tokens use CSS clamp() for fluid scaling from 320px-1200px viewport. Global styles include dvh body height, touch target minimums for coarse pointers, and progressive content padding reduction at 768px and 480px breakpoints.
