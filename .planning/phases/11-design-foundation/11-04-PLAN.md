<task type="auto">
  <name>Task 2: Verify WCAG AA contrast ratios</name>
  <files></files>
  <action>
Verify the specific color combinations meet WCAG AA 4.5:1 contrast ratio for normal text.

**Expected color pairings from design tokens:**
| Element | Foreground | Background | Required Ratio |
|---------|------------|------------|----------------|
| Body text | #1a1a1a (--color-text) | #fafafa (--color-background) | 4.5:1 |
| Links | #0066cc (--color-accent) | #fafafa (--color-background) | 4.5:1 |
| Card text | #1a1a1a (--color-text) | #ffffff (--color-surface) | 4.5:1 |
| Muted text | #666666 (--color-text-muted) | #fafafa (--color-background) | 4.5:1 |

**Manual verification using WebAIM Contrast Checker:**

1. Open https://webaim.org/resources/contrastchecker/
2. Test each pairing:
   - Body text: FG=#1a1a1a, BG=#fafafa → Expected: ~16.0:1 (PASS)
   - Links: FG=#0066cc, BG=#fafafa → Expected: ~5.3:1 (PASS)
   - Card text: FG=#1a1a1a, BG=#ffffff → Expected: ~17.4:1 (PASS)
   - Muted text: FG=#666666, BG=#fafafa → Expected: ~5.7:1 (PASS)

3. If any pairing fails 4.5:1:
   - Document the failing pair
   - Suggest adjustment (darker foreground or lighter background)

**Note:** The actual color values are defined in src/styles/variables.css. If they differ from above, extract the actual values and test those.

To extract actual colors:
```bash
grep -E "(--color-text|--color-background|--color-accent|--color-surface|--color-text-muted)" src/styles/variables.css
```

   - Click nav links - they should work (texture not blocking)
   - Click on a publication/talk/media card - should navigate
   - Hover over cards - shadow should increase, slight lift

4. Check card styling:
   - Cards have subtle shadow at rest
   - Hover shows increased shadow
   - Border changes to blue on hover

5. Test reduced motion (in DevTools):
   - Open DevTools > Rendering > Emulate CSS media feature
   - Set prefers-reduced-motion to "reduce"
   - Hover over cards - shadow changes but NO movement
   - Scroll - should be instant, not smooth

6. Contrast verification (already done in Task 2, but visual confirmation):
   - Body text clearly readable against off-white background
   - Links distinguishable and readable
   - Card text readable on white cards
   - Any muted/secondary text still readable
