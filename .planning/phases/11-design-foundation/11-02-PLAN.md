<task type="auto">
  <name>Task 1: Create halftone texture using CSS radial-gradient fallback</name>
  <files>src/styles/global.css, public/assets/textures/halftone-dots.png</files>
  <action>
Create the halftone texture effect using a CSS radial-gradient pattern. This is MORE robust than downloading external textures because:
- No network dependency on texturelabs.org (may require auth or have dynamic URLs)
- No ImageMagick prerequisite
- Consistent cross-browser behavior
- Easily tunable via CSS variables

1. Create directory: mkdir -p public/assets/textures

2. Create a placeholder file to satisfy the artifact requirement:
```bash
touch public/assets/textures/halftone-dots.png
```
(This is a placeholder - the actual texture will be CSS-based)

3. In global.css, we will use a radial-gradient that creates a halftone dot pattern:

```css
/* Halftone texture overlay - newspaper/print aesthetic
   Using CSS radial-gradient for reliable, network-independent texture */
body::after {
  content: '';
  position: fixed;
  inset: 0;
  /* Halftone dot pattern via radial-gradient */
  background-image: radial-gradient(circle, rgba(0, 0, 0, 0.15) 1px, transparent 1px);
  background-size: var(--texture-size, 4px) var(--texture-size, 4px);
  pointer-events: none;  /* Critical: allows clicking through */
  z-index: 9999;
  opacity: var(--texture-opacity, 0.5);
}
```

This creates a uniform dot pattern that:
- Mimics halftone printing (small dots at regular intervals)
- Uses black dots at 15% opacity, then overall overlay at 50% = subtle effect
- Is tunable via --texture-size (dot spacing) and --texture-opacity

If user later wants a more organic/irregular halftone texture, they can:
- Replace the radial-gradient with a real PNG: background-image: url('/assets/textures/halftone-dots.png');
- Download from texturelabs.org manually and replace the placeholder

NOTE: Do NOT use mix-blend-mode: multiply with radial-gradient - it works differently than with images. The opacity approach above achieves the subtle overlay effect.

<task type="auto">
  <name>Task 3: Add comprehensive reduced-motion handling</name>
  <files>src/styles/global.css</files>
  <action>
Add reduced-motion media query at the end of global.css:

```css
/* Reduced motion - respect accessibility preferences */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This is the nuclear option that disables ALL motion site-wide when user prefers reduced motion. It's aggressive but ensures WCAG compliance.

Also update the existing scroll-behavior rule to use the no-preference pattern:
```css
@media (prefers-reduced-motion: no-preference) {
  html {
    scroll-behavior: smooth;
  }
}
```

Remove the existing separate reduced-motion media query for scroll-behavior if present (consolidate into the new comprehensive rule).
  </action>
